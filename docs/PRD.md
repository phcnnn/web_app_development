# 食譜收藏夾系統 PRD

## 1. 專案概述

- **背景與動機**
  日常生活中常常會看到不錯的食譜，或者自己研發出好吃的料理，但往往散落在不同的筆記或書籤中，難以統整。本系統希望提供一個專屬的數位空間，方便集中管理、紀錄與檢索個人的食譜內容，包含食材明細與詳細步驟。
- **目標用戶**
  我自己（作為系統唯一使用者與管理者）。
- **核心價值主張**
  簡單、直覺的食譜管理與檢索功能，以及能根據現有食材推薦或檢索食譜的能力，讓料理過程更有效率、解決不知道吃什麼的煩惱，並達到清冰箱的目的。

## 2. 功能需求

- **1. 新增食譜**
  - 作為使用者，我希望可以新增包含標題、簡介、所需時間等基本資訊的食譜，以便把喜歡的料理紀錄下來。
- **2. 製作步驟與材料紀錄**
  - 作為使用者，我希望可以為每份食譜列出詳細的「材料清單（含份量）」與「製作步驟」，以便在料理時能夠按部就班地參照執行。
- **3. 分類管理**
  - 作為使用者，我希望可以幫食譜加上「分類」（如：中式、西式、甜點、早餐等），以便日後能快速找到特定類型的料理。
- **4. 搜尋食譜**
  - 作為使用者，我希望可以透過關鍵字搜尋食譜的名稱或材料，以便在忘記特定食譜名稱時，能迅速找到目標結果。
- **5. 根據食材製作食譜**
  - 作為使用者，我希望可以輸入目前手邊現有的食材，系統能篩選出符合這些關鍵食材的推薦食譜，以便我能快速決定今天的菜色並解決過期食材。

## 3. 非功能需求

- **技術限制**
  - 應用程式架構：後端使用 Python Flask，前端渲染使用 Jinja2 模板引擎搭配基本 HTML/CSS。
  - 資料庫：SQLite 作為輕量級的資料存儲。
- **效能與安全考量**
  - 以單人使用為主，重視系統反應速度與操作流暢度。
  - 針對使用者輸入表單進行基本的防止 SQL Injection（使用 ORM 或參數化查詢）以及 XSS （Jinja2 預設會進行 HTML 逸出）防護。

## 4. MVP 範圍

- **Must Have (必須有)**
  - 新增食譜（包含食譜名稱、所需材料、步驟說明）。
  - 檢視系統中所有的食譜列表。
  - 查看單一食譜的詳細內容。
  - 基礎分類功能（可在新增時選擇一個分類）。
- **Should Have (應該有)**
  - 關鍵字搜尋（針對食譜名稱或食材）。
  - 編輯與刪除已建立的食譜。
- **Nice to Have (加分項)**
  - 根據手邊輸入的「多項食材」進行交集查詢與配對功能的推薦。
  - 圖片上傳（為食譜加上成品封面圖）。
